## 📱 管家会收到什么

### 1. 分析报告
每次运行分析时，管家会收到：
- 实时行情数据
- 持仓盈亏情况
- 技术指标分析
- 操作建议

### 2. 操作提醒
当触发以下情况时，管家会收到提醒：
- 🔴 止盈提醒（达到15%）
- 🔴 止损提醒（达到-8%）
- 🟡 持仓时间提醒（接近30天）
- 📡 技术信号提醒

### 3. 每日报告
可以配置定时发送：
- 早上9:00：每日分析报告
- 下午3:00：市场更新

## ⚙️ 高级配置

### 修改发送内容类型

编辑 `butler_config.py`：

```python
"send_types": [
    "daily_report",      # 每日报告
    "alert",             # 操作提醒
    "analysis",          # 分析报告
    "market_update",     # 市场更新
    "position_update"    # 持仓更新
]
```

注释掉不需要的类型即可禁用。

### 修改发送时间

```python
"schedule": {
    "daily_report": "09:00",  # 每日报告发送时间
    "market_update": "15:00"  # 市场更新发送时间
}
```

### 启用/禁用管家

```python
BUTLER_CONFIG = {
    "enabled": True,  # True启用，False禁用
    ...
}
```

## 🔧 故障排除

### 问题1：管家收不到消息

**检查清单：**
- [ ] Chat ID是否正确
- [ ] Bot是否有管理员权限
- [ ] 网络连接是否正常
- [ ] 配置是否已保存

**解决方法：**
```bash
# 1. 测试配置
python3 setup_butler.py
# 选择选项3测试

# 2. 检查日志
cat ~/.stock_monitor/002837/cron.log

# 3. 手动测试
python3 butler_config.py
```

### 问题2：消息格式错误

**可能原因：**
- 消息内容包含特殊字符
- 消息过长

**解决方法：**
- 检查消息内容
- 分段发送长消息

### 问题3：重复发送

**可能原因：**
- 定时任务配置错误
- 发送记录未保存

**解决方法：**
- 检查crontab配置
- 检查alert_sent记录
