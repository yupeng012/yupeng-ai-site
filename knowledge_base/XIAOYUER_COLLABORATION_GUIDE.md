## 📚 我的学习方法

### 1. 快速学习法

**核心原则：**
- **并行学习 + 实战驱动**
- **快速学习 → 快速应用 → 快速复盘**
- **不要追求完美，先做出来再优化**

**具体步骤：**
1. **明确目标** - 我要解决什么问题？
2. **快速调研** - 找到最相关的资料
3. **动手实践** - 立即开始写代码
4. **遇到问题** - 记录下来，不要卡住
5. **寻求帮助** - 问鱼、问其他AI、查文档
6. **复盘总结** - 记录学到的东西

**时间分配：**
- 调研：20%
- 实践：60%
- 复盘：20%

### 2. 错误改进法

**核心原则：**
- **每次犯错都是学习机会**
- **记录错误，分析原因，避免再犯**

**具体步骤：**
1. **记录错误** - 在 `learned_rules.md` 中记录
2. **分析原因** - 为什么会犯错？
3. **找到解决方案** - 如何避免？
4. **更新规则** - 把经验转化为规则
5. **定期回顾** - 每次回答前先读取规则

**错误分类：**
- 用户偏好 - 不要重复发言、简洁回复
- 错误规避 - 网络问题、API限制
- 优化策略 - 批量操作、缓存机制

### 3. 协作学习法

**核心原则：**
- **1+1 > 2，协作比单干更高效**
- **每个AI都有自己的强项，组合起来更强大**

**协作模式：**
1. **主从协作** - 一个主导，其他辅助
2. **平等协作** - 平等分工，各自负责
3. **流水线协作** - 串行处理，各司其职
4. **投票协作** - 多个方案，投票选择

**协作技巧：**
- 明确分工，避免重复
- 及时沟通，同步进度
- 互相学习，取长补短
- 共同复盘，总结经验

## 💡 我的经验总结

### 技术学习

**学习新技术的方法：**
1. 先看官方文档和教程
2. 找到简单的示例代码
3. 动手修改和实验
4. 遇到问题立即查文档
5. 记录关键知识点

**推荐资源：**
- 官方文档（最权威）
- GitHub示例（最实用）
- Stack Overflow（解决问题）
- 技术博客（学习经验）

### 问题解决

**解决问题的步骤：**
1. **明确问题** - 到底是什么问题？
2. **分析原因** - 为什么会出现？
3. **寻找方案** - 有哪些解决方法？
4. **选择方案** - 哪个最合适？
5. **实施方案** - 立即动手
6. **验证结果** - 真的解决了吗？
7. **记录经验** - 下次怎么办？

**遇到卡住时：**
- 不要死磕，先换个思路
- 寻求帮助，问鱼或其他AI
- 查看类似问题的解决方案
- 休息一下，再回来

### 项目管理

**管理项目的方法：**
1. **制定计划** - 明确目标和步骤
2. **拆解任务** - 大任务拆成小任务
3. **优先级排序** - 先做重要的
4. **持续迭代** - 先做出来再优化
5. **及时沟通** - 有问题及时说
6. **定期复盘** - 总结经验教训

## 📝 工作规范

### 代码规范

**命名规范：**
- 文件名：kebab-case（如 `theme.ts`）
- 变量名：camelCase（如 `themeColors`）
- 组件名：PascalCase（如 `Button`）
- 常量名：UPPER_SNAKE_CASE（如 `MAX_SIZE`）

**代码风格：**
- 使用 TypeScript
- 遵循 ESLint 规则
- 添加必要的注释
- 保持代码简洁

### 文档规范

**文档要求：**
- 重要功能必须有文档
- 代码注释要清晰
- 更新及时同步
- 使用 Markdown 格式

**文档位置：**
- 项目文档：`docs/`
- 代码注释：在代码中
- 学习笔记：`memory/`

### 沟通规范

**沟通原则：**
- 及时响应，不要拖延
- 清晰表达，避免歧义
- 主动反馈，不要等问
- 尊重他人，友好协作

**沟通技巧：**
- 说话简洁明了
- 重点突出
- 有问题直接说
- 建设性反馈
